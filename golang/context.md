#  Context в golang

## Описание

Context - это инструмен управления жизненым циклом операций, передачи сигналов отмены и хранения данных внутри области запроса.
Для чего нужен context? Для отмены операций, ограничивает время выполнения задачи, хранит специфичные запросы данные. 

## Как его используюти и его синтаксис

Context - передается как первый аргумент в функции. Пример: `func DoSomething(ctx context.Context, ...)`.
### Основные функции создания:
- `context.Background()` - пустой корень с которого обычно все начинается.
- `context.TODO()` - используется, если вы еще не решили какой контекст использовать.
- `context.WithCancel(parent)` - создает контекст, контекст который можно отменить вручную через `cancel()`.
- `context.WithTimeout(parent, duration)` - автоматически отменяется через указанное время.
- `context.WithValue(parent, key, value)` - добавляет в контекст данные. 

## Примеры

Разберем некоторые примеры использования context.

```go
    func main() {
        ctx, cancel := context.WithCancel(context.Background())

        wg := sync.WaitGroup{}

        wg.Add(1)
        go worker(ctx, &wg)


        time.Sleep(5 * time.Second)
        cancel()
        wg.Wait()
    
    }

    func worker(ctx context.Context, wg *sync.WaitGroup) {
        defer wg.Done()

        ticker := time.NewTicker(1 * time.Second)
        defer ticker.Stop()

        for i := 0; i < 5; i++ {
            select {
            case <-ctx.Done():
                fmt.Println("context: ", ctx.Err())
                return
            case <-ticker.C:
                fmt.Println("working: ", i)
            }
        }
    }
   ```
   Что здесь происходит, мы создаем контекст который отменяется когда у нас проходит 5 - секунд, как он отменяется? Когда мы используем `context.WithCancel(context.Background)` то возвращается два значения первый это сам контекст второй это отмена контекста, и отмену мы можем вызывать вручную где хотим.
   Далее мы через горунтину вызываем нашу функцию `worker` - которая как раз принимает наш контекст. Для чего он принимает контект, а для того чтобы понять, прошло 5 - секунд или нет.
   
   Далее внутри самой функции мы запускаем цикл который будет итерироваться не более 5 раз, и внути мы через `select` - проверяем что у нас лежит в контексте и в нашем тикере, если контекст был отменен, а это можно узнать через `ctx.Done()` - то есть когда у нас отменяется контекст `ctx.Done()` - что контекст был отменен. Если он был отменен то мы показываем это, а если контекст пока не отменен то мы просто показываем через каждую секуду нашего воркера.
