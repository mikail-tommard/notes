# Stack or Heap

## Описание

Разберем с вам что такое `stack` и `heap` и как их использовать в golang, так же понять для чего они нужны.

## Stack

`Stack` - это память для короткоживущих локальных данных в функциях. Еще так же говорят что это `стэк` - вызовов. Когда функция вызывается обычно для нее выделятся память под: локальные переменные, параметры и т.д. Когда функция закончилась память освобождается. То есть `stack` - это память: 1. временная 2. быстрая 3. хорошо подходит для локальных значений.

## Пример

```golang
func f() {
  x := 10
}
```

Данная функция имеет локальную переменную, которая долго не живет и ее память освободиться как функция закончить свою работу.
Даже если мы напишем вот так:

```golang
func f() int {
  x := 10
  return x
}
```

И это тоже скорее всего будет в `stack` - так как память будет освобождаться как функция закончить свою работу. Даже если функция что-то возвращает это не значит что функци указывает на какую то память которая должна жить долго.
