# Cat-file

## Описание

При работе с git, чаще всего мы все видим на поверхности. Но как все устроено внутри git? Git - внутри себя хранит все в виде объектов(коммиты, деревья, блобы, теги).
И чтобы посмотреть под капот мы можем использовать команду `git cat-file`.
Сама команда `git cat-file` не используется и не нужна в повседневной работе с Git, но она не заменима если вы хотите узнать подробности и вообще узнать Git внутри или отлаживать какие-то необычные ситуации.

## Какие типы объектов есть в Git

- blob - содержимаое файла(бинарник)
- tree - дерево или же каталог, показывает список файлов и каталогов и ссылки на их содержимое
- commit - коммит, тут уже показывается корневое дерево и метаданные автора
- tag - тег, аннотированный тег в виде объекта, который указывает на коммит
