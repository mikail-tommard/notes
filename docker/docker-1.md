# Что такое Docker и как он работает

## Описание

Docker - это программа которая помогает запускать приложения в любой среде.
Общая идея такая: у тебя есть приложение для него нужны определенные библиотеки и т.д. Ты все настроил на свои комптьютере и у тебя все будет работать, но на другом компьютере уже работать не будет. Docker - упаковывает это все вместе и позволяет наше приложение запускать на любом другом компьютере без каких либо проблем с настройкой.

## Что такое контейнер

Контейнер - это изолированная среда внутри которой работает наше приложение.
Контейнер можно представить как коробку, внутри которой живет: само приложение, все его зависимости и т.д.
Внутри контейнера приложение как будто живет в своем собственном мире, а снаружи у тебя может быть один компьютер

## Пример

Возмем на примере базы данных SQL. Базу данных мы можем запустить с помощью dekstop программ для SQL. Но так же можно запустить нашу базу данных в Docker и уже через контейнер работать с ним.
Для начала следует создать docker-compose.yml - и там уже мы опишем наш сервис postgreSQL

```yaml
services:
  postgres:
    image: postgres:latest
    container_name: postgres
    restart: always
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: postgres
    ports:
      - '5433:5432'
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

Сначала мы описываем services: тут описывается контейнеры которые у нас буду запускаться, далее уже идет имя сервиса то есть postgres: внутри мы описываем наш образ так же имя контейнера. После что будет когда контейнер остановиться, в нашем случае он попытается его снова запустить. Environment: здесь описывается зависимости нашего сервиса, далее мы говорим на каких портах он будет запущен ports: "5433:5432" - первый порт 5433 говорит на каком порту он будет запушен внутри контейнера, а 5432 на какой порт мы сможем у себя на компьютере к нему обращаться. Volumes - тут означает что наши данные postreSQL будут отдельно сохраняться в Docker volumes.
Как нам теперь запустить наш сервис, сначала надо запустить dekstop приложение Docker и после в терминали уже написать `docker-compose up -d`.
